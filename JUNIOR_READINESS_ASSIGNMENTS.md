# Junior Readiness Assignments

Этот файл - не просто список упражнений, а минимальный набор работ, после которых уже имеет смысл пробовать отклики на junior/trainee позиции.

Важно:

- не просто "решить", а довести до состояния, когда ты можешь объяснить решение
- каждое решение должно корректно работать на плохом вводе, если это уместно
- каждую работу желательно приносить на ревью

## Минимальный стандарт качества для любого задания

Перед сдачей проверь:

- код компилируется с `-Wall -Wextra -pedantic`
- нет очевидно лишнего debug output
- граничные случаи проверены руками
- имена переменных и функций читаемы
- ошибки не маскируются под корректный результат

## Этап 1. База языка и дисциплина ввода

Цель этапа:

- перестать путать "прочиталось" и "похоже на правду"
- научиться видеть граничные случаи до ревью

### Задание 1

Калькулятор двух чисел:

- поддержать `+ - * /`
- обработать невалидный формат
- обработать деление на ноль
- не смешивать код ошибки и корректный результат

### Задание 2

Статистика массива:

- прочитать `n`
- прочитать `n` чисел
- вывести `min`, `max`, `average`, `count of even`
- запретить `n <= 0`
- запретить нехватку или избыток данных

### Задание 3

Разворот массива:

- прочитать `n`
- прочитать `n` чисел
- вывести массив в обратном порядке
- сделать отдельную функцию для реверса

### Задание 4

Палиндром строки:

- прочитать строку
- определить, палиндром ли она
- сделать 2 версии:
  - чувствительную к регистру
  - нечувствительную к регистру и пробелам

### Задание 5

Подсчет частот слов:

- прочитать строку текста
- разбить на слова
- вывести частоту каждого слова
- использовать `std::unordered_map`

## Критерий закрытия этапа 1

Ты без подсказок:

- валидируешь вход
- отличаешь ошибку чтения от значения `0`
- не допускаешь off-by-one в простых циклах
- умеешь объяснить, почему передаешь `vector` или `string` по `const&`

## Этап 2. STL и разбиение логики

Цель этапа:

- уйти от стиля "все в main"
- начать писать маленькие, но структурные программы

### Задание 6

Менеджер книг:

- структура `Book`
- хранение книг в `std::vector`
- поиск по названию
- поиск по автору
- вывод списка книг

### Задание 7

Подсчет статистики текста:

- количество строк
- количество слов
- количество символов
- самое частое слово

Разделить код на функции:

- чтение
- разбор
- вычисление
- вывод

### Задание 8

Удаление дублей:

- прочитать массив
- удалить дубли
- сохранить порядок первого вхождения

### Задание 9

Группировка строк по первой букве:

- вход: список слов
- выход: структура, где слова сгруппированы по первой букве

Использовать:

- `std::map` или `std::unordered_map`
- `std::vector`

### Задание 10

Todo CLI без сохранения:

- add
- list
- done
- remove

Разделить хотя бы на:

- модель данных
- команды
- вывод

## Критерий закрытия этапа 2

Ты:

- используешь STL по делу
- выносишь логику из `main`
- перестаешь плодить лишние классы там, где хватает функций
- можешь объяснить выбор контейнера

## Этап 3. Алгоритмы и точность мышления

Цель этапа:

- начать стабильно решать типовые interview-style задачи начального уровня

### Задание 11

Two Sum

- дать решение
- назвать сложность
- сделать тестовые кейсы

### Задание 12

Valid Parentheses

- использовать стек
- обработать пустой ввод

### Задание 13

Merge Two Sorted Arrays

- без бессмысленных лишних копий
- с объяснением сложности

### Задание 14

Maximum sum of fixed-size window

- реализовать sliding window
- объяснить, почему это лучше наивного решения

### Задание 15

Binary search

- классический бинарный поиск по отсортированному массиву
- отдельно объяснить условие остановки

## Критерий закрытия этапа 3

Ты:

- сначала выбираешь структуру данных, потом кодишь
- можешь назвать сложность решения
- меньше ошибаешься в условиях циклов и границах диапазонов

## Этап 4. Инженерный минимум под junior C++

Цель этапа:

- перестать быть "человеком с одним .cpp файлом"

### Задание 16

Возьми одну из прошлых задач и оформи ее как мини-проект:

- `src/`
- `include/`
- `CMakeLists.txt`
- `README.md`

### Задание 17

Добавь к этому проекту:

- минимум 5 тестов
- инструкцию по сборке
- инструкцию по запуску

### Задание 18

Добавь строгую сборку:

- C++17
- warning flags
- debug/release

### Задание 19

Сделай ревизию проекта:

- убрать дублирование
- убрать длинные функции
- переименовать неудачные сущности
- отделить ввод от логики

## Критерий закрытия этапа 4

Ты:

- умеешь собрать проект с нуля
- можешь объяснить структуру проекта
- не путаешь вычисление, ввод и вывод

## Этап 5. Финальный отборочный проект

Цель этапа:

- получить проект, который уже не стыдно показывать

### Выбери один проект

1. `task-tracker-cli`
2. `log-analyzer`
3. `file-indexer`
4. `student-records-manager`

### Обязательные требования

- CMake
- несколько `.cpp/.h` файлов
- README
- обработка ошибок
- тесты
- STL-контейнеры
- осмысленные имена и структура

### Что ты должен уметь по нему рассказать

- что делает проект
- как он устроен
- где основная логика
- какие структуры данных выбраны
- как он собирается
- какие ограничения у него есть

## Когда можно пробовать junior-отклики

Когда закрыты:

- все задания этапов 1-4
- финальный проект этапа 5

И когда ты можешь:

- без паники решать простые прикладные задачи
- объяснять свои решения
- не допускать очевидных ошибок валидации и границ на каждом шаге
- работать с Git, CMake и Linux CLI на базовом уровне
