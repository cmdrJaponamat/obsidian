## Правила прохождения

- Не пропускай практику.
- На каждом модуле делай минимум одну самостоятельную реализацию без подсказок.
- Если код не собирается, модуль считается не закрытым.
- Раз в неделю перечитывай собственный код и переписывай хотя бы один кусок лучше.

## Минимальный результат трека

После завершения трека ты должен уметь:

- писать консольные утилиты и маленькие сервисные программы на C++
- пользоваться `std::vector`, `std::string`, `std::unordered_map`, `std::optional`, `std::variant`
- понимать lifetime объектов и не бояться RAII
- собирать проект через CMake
- читать ошибки компилятора и отладчика
- писать простые тесты
- объяснить разницу между raw pointer, reference, `unique_ptr`, `shared_ptr`
- решить набор типовых задач на строки, массивы, map/set, стек/очередь
