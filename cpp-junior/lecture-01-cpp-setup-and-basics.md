# Лекция 1. Что такое программа на C++ и из чего она состоит

## Как читать эту лекцию

Это не краткий конспект, а объяснение с нуля. Если какая-то часть кажется "слишком простой", это нормально: цель не пропустить фундамент, на котором потом держатся память, ссылки, RAII и STL.

## Зачем вообще нужен C++

C++ нужен там, где важны:

- скорость выполнения
- контроль над памятью и временем жизни объектов
- предсказуемость работы программы
- возможность писать системные, сетевые, игровые и инфраструктурные программы

Для junior сейчас важнее не "знать весь язык", а понять:

- что такое исходный код, компиляция и запуск
- как устроены переменные и функции
- как передаются данные
- почему в C++ так много внимания к памяти и lifetime

## Что такое программа на C++

Программа на C++ обычно состоит из:

- исходных файлов `.cpp`
- заголовочных файлов `.h` или `.hpp`
- компилятора
- линковщика
- стандартной библиотеки

Простейшая программа:

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, C++\n";
    return 0;
}
```

Разберем ее медленно.

### `#include <iostream>`

Это директива препроцессора. Она вставляет содержимое заголовка `iostream`, где объявлены средства ввода и вывода, например `std::cout`.

### `int main()`

`main` - специальная функция, с которой начинается выполнение программы.

- `int` означает, что функция возвращает целое число
- обычно `0` означает успешное завершение
- ненулевой код возврата часто означает ошибку

### `std::cout`

Это стандартный поток вывода. Через него программа пишет текст в терминал.

### `\n`

Это перевод строки.

## Что происходит после того, как ты написал код

Когда ты запускаешь сборку, происходит не "магия", а довольно конкретная цепочка шагов.

### Шаг 1. Компилятор читает исходный код

Он проверяет:

- синтаксис
- типы
- корректность вызовов функций
- наличие нужных объявлений

Если ты написал:

```cpp
int x = "hello";
```

компилятор остановится с ошибкой, потому что строка не может быть неявно присвоена `int`.

### Шаг 2. Получаются объектные файлы

Каждый `.cpp` файл обычно компилируется отдельно в промежуточный результат.

### Шаг 3. Линковщик собирает итоговую программу

Он соединяет:

- твои объектные файлы
- стандартную библиотеку
- при необходимости внешние библиотеки

Если ты объявил функцию, но не дал реализацию, ошибка может проявиться именно на этапе линковки.

## Типы данных: что это и зачем они нужны

Тип отвечает на вопрос: "какие значения может хранить переменная и какие операции над ней допустимы".

Примеры:

- `int` - целые числа
- `double` - числа с плавающей точкой
- `bool` - `true` или `false`
- `char` - символ
- `std::string` - строка

```cpp
int age = 20;
double temperature = 36.6;
bool is_ready = true;
char grade = 'A';
std::string name = "Ivan";
```

### Почему типы важны

В C++ типы очень сильно влияют на:

- объем памяти
- допустимые операции
- правила копирования и передачи
- производительность

Позже ты увидишь, что выбор типа - это уже инженерное решение, а не просто синтаксис.

## Переменная: что это на самом деле

Переменная - это именованная область памяти, в которой хранится значение определенного типа.

Когда ты пишешь:

```cpp
int count = 5;
```

ты сообщаешь компилятору:

- нужно место для целого числа
- имя этого объекта `count`
- начальное значение равно `5`

### Почему лучше инициализировать сразу

Потому что неинициализированные данные часто ведут к ошибкам.

Плохо:

```cpp
int count;
```

Лучше:

```cpp
int count = 0;
```

## `const`: не просто украшение

`const` означает: этот объект нельзя менять после инициализации.

```cpp
const int days_in_week = 7;
```

### Что это дает

- защищает от случайного изменения
- делает намерение кода понятнее
- позволяет строить более безопасные интерфейсы

Если ты видишь `const`, то сразу понимаешь: здесь значение только читают.

## Область видимости

Область видимости определяет, где имя доступно.

```cpp
int main() {
    int x = 10;
    {
        int y = 20;
    }
}
```

Здесь:

- `x` доступен внутри `main`
- `y` доступен только внутри внутреннего блока

Когда блок заканчивается, объект обычно уничтожается. Позже это станет критически важно для понимания lifetime.

## Выражения, операции и преобразования типов

```cpp
int a = 5;
int b = 2;
double result = a / b;
```

Новичок часто ожидает `2.5`, но получит `2`.

Почему:

- `a` и `b` имеют тип `int`
- деление двух `int` дает целый результат
- только потом он приводится к `double`

Правильнее:

```cpp
double result = static_cast<double>(a) / b;
```

Из этого сразу видно важное правило: типы влияют не только на хранение, но и на вычисления.

## Ветвления

```cpp
if (age >= 18) {
    std::cout << "Adult\n";
} else {
    std::cout << "Minor\n";
}
```

Здесь программа выбирает одно из двух направлений выполнения.

### Что важно

- условие должно давать булево значение
- код внутри `{}` образует отдельный блок
- чем проще условие, тем легче читать

Плохо:

```cpp
if ((a > 10 && b < 20) || (!flag && c == 7)) {
```

Если условие сложное, его лучше вынести в отдельную булеву переменную с понятным именем.

## Циклы

### `for`

```cpp
for (int i = 0; i < 5; ++i) {
    std::cout << i << '\n';
}
```

Этот цикл полезен, когда:

- есть счетчик
- заранее понятен общий шаблон повторения

### `while`

```cpp
int value = 3;
while (value > 0) {
    std::cout << value << '\n';
    --value;
}
```

Этот вариант удобнее, когда повторение идет "пока выполняется условие".

## Функции: зачем они нужны

Функция - это именованный кусок поведения.

```cpp
int square(int value) {
    return value * value;
}
```

### Что дают функции

- уменьшают дублирование
- делают код читаемым
- позволяют тестировать логику отдельно
- помогают разделять ответственность

Если программа целиком написана в `main`, это почти всегда плохой знак.

## Передача аргументов в функцию

Это критически важная тема. Здесь многие новички видят синтаксис, но не понимают смысл.

### Передача по значению

```cpp
void print_number(int value) {
    std::cout << value << '\n';
}
```

Что происходит:

- функция получает собственную копию аргумента
- изменение `value` внутри функции не меняет исходную переменную снаружи

Это просто и безопасно, если тип маленький:

- `int`
- `double`
- `bool`

### Передача по ссылке

```cpp
void increment(int& value) {
    ++value;
}
```

Что это значит:

- `value` внутри функции становится другим именем для исходного объекта
- копия не создается
- если изменить `value`, изменится исходная переменная

Это полезно, когда функция должна менять объект.

### Передача по `const` ссылке

```cpp
void print_name(const std::string& name) {
    std::cout << name << '\n';
}
```

Вот здесь как раз тот смысл, который часто проговаривают слишком быстро.

Что это дает:

1. Не создается копия строки.
2. Функция получает доступ к исходному объекту.
3. `const` запрещает менять этот объект через данный параметр.

Почему это важно:

- `std::string` и `std::vector` могут быть большими
- копирование большого объекта стоит времени и памяти
- если функция должна только читать данные, копия не нужна

То есть `const std::string&` означает:

- "дай мне строку"
- "я не буду ее копировать без необходимости"
- "я не буду ее менять"

Это один из самых важных паттернов современного C++.

### Когда что использовать

- маленькие типы: часто по значению
- большие объекты только для чтения: часто `const&`
- объекты, которые надо изменить: `&`

Это не абсолютный закон, но очень хороший старт.

## `std::string`

Это стандартный тип строки.

```cpp
std::string text = "hello";
```

Что он дает по сравнению с C-строками:

- удобный и безопасный интерфейс
- хранит размер
- умеет расти и изменяться
- лучше вписывается в STL

## `std::vector`

Это динамический массив из стандартной библиотеки.

```cpp
std::vector<int> numbers = {1, 2, 3};
numbers.push_back(4);
```

Что значит "динамический":

- размер можно менять во время выполнения
- память под элементы управляется самим контейнером

Для junior это один из основных контейнеров.

### Как обходить `vector`

Классический цикл:

```cpp
for (std::size_t i = 0; i < numbers.size(); ++i) {
    std::cout << numbers[i] << '\n';
}
```

Range-based `for`:

```cpp
for (int value : numbers) {
    std::cout << value << '\n';
}
```

Важно уточнение: range-based `for` появился в стандарте C++11, не в C++20.

### Как работает `for (int value : numbers)`

Смысл такой:

- контейнер предоставляет начало и конец последовательности
- цикл проходит по элементам
- на каждой итерации `value` получает очередной элемент

Но здесь есть тонкость:

```cpp
for (int value : numbers)
```

`value` - это копия элемента.

Если элементы тяжелые, лучше:

```cpp
for (const int& value : numbers)
```

или чаще для пользовательских типов:

```cpp
for (const std::string& value : words)
```

Что это дает:

- нет копирования каждого элемента
- элемент только читается

А если нужно менять элементы:

```cpp
for (int& value : numbers) {
    value *= 2;
}
```

## Ввод и вывод

```cpp
int x = 0;
if (!(std::cin >> x)) {
    std::cerr << "Invalid input\n";
    return 1;
}
```

Это лучше, чем просто писать `std::cin >> x;` и надеяться, что пользователь всегда вводит корректные данные.

## Хороший маленький пример

```cpp
#include <iostream>
#include <string>
#include <vector>

int sum(const std::vector<int>& numbers) {
    int result = 0;
    for (int value : numbers) {
        result += value;
    }
    return result;
}

int main() {
    std::vector<int> numbers = {1, 2, 3, 4, 5};
    std::cout << "Sum: " << sum(numbers) << '\n';
    return 0;
}
```

Что здесь уже хорошо:

- логика вынесена в функцию
- `vector` передается по `const&`
- имена осмысленные
- программа маленькая, но структурная

## Типичные ошибки новичка

- писать всю программу в `main`
- не понимать, когда происходит копирование
- не использовать `const`
- путать "ссылка" и "копия"
- использовать `int` везде без размышления
- не проверять ввод

## Что нужно вынести из этой лекции

- программа на C++ проходит стадии компиляции и линковки
- типы влияют и на данные, и на вычисления
- функции нужны не для красоты, а для структуры
- `const&` - это способ читать объект без копии и без права изменения
- range-based `for` в C++ существует давно, и важно понимать, копируешь ты элементы или берешь ссылки
