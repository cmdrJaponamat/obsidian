# Лекция 1. Среда, базовый синтаксис и модель программы

## Зачем это изучать

На junior собеседовании по C++ часто быстро становится видно, писал ли кандидат реальный код или только видел язык на занятиях. Первая цель не в том, чтобы знать "весь C++", а в том, чтобы стабильно писать и запускать маленькие программы без хаоса.

## Что такое C++ в практическом смысле

C++ - это компилируемый язык общего назначения, который часто используют там, где важны:

- производительность
- контроль над памятью и временем жизни объектов
- системное программирование
- игры, embedded, backend, сетевые и инфраструктурные сервисы

Для junior важнее не "все возможности языка", а вот эти основы:

- понимать, как код превращается в исполняемый файл
- уметь читать ошибки компиляции
- понимать типы, область видимости и функции
- писать простой читаемый код

## Как выглядит жизненный цикл программы

Упрощенно:

1. Ты пишешь исходный код в `.cpp` и `.h`.
2. Компилятор проверяет синтаксис и типы.
3. Линковщик собирает итоговый исполняемый файл.
4. Программа запускается, создает объекты, выполняет логику, завершает работу.

Минимальный пример:

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, C++\n";
    return 0;
}
```

Здесь:

- `#include <iostream>` подключает средства ввода/вывода
- `int main()` - точка входа
- `std::cout` пишет в стандартный вывод

## Базовые типы и переменные

Чаще всего в начале используются:

- `int`
- `long long`
- `double`
- `bool`
- `char`
- `std::string`

Пример:

```cpp
int age = 21;
double rating = 4.7;
bool is_ready = true;
std::string name = "Ivan";
```

### Что важно сразу запомнить

- тип переменной определяет, какие значения и операции допустимы
- лучше инициализировать переменную сразу
- имена должны объяснять смысл, а не только тип

Плохо:

```cpp
int a = 10;
```

Лучше:

```cpp
int max_connections = 10;
```

## `const` как базовая дисциплина

Если значение не должно меняться, помечай его `const`.

```cpp
const int days_in_week = 7;
```

Это полезно потому, что:

- код становится понятнее
- компилятор помогает не менять значение случайно
- ты начинаешь четче понимать, где данные только читаются

## Ветвления и циклы

### `if`

```cpp
if (age >= 18) {
    std::cout << "Adult\n";
} else {
    std::cout << "Minor\n";
}
```

### `for`

```cpp
for (int i = 0; i < 5; ++i) {
    std::cout << i << '\n';
}
```

### `while`

```cpp
int value = 3;
while (value > 0) {
    std::cout << value << '\n';
    --value;
}
```

## Функции

Функция нужна, чтобы вынести отдельную законченную операцию.

```cpp
int square(int value) {
    return value * value;
}
```

Преимущества:

- меньше дублирования
- проще тестировать
- проще читать

### Хороший стиль функции

- одна функция делает одну понятную вещь
- имя функции описывает действие
- аргументы передаются осознанно

Плохо:

```cpp
void process() {
    // читает ввод, считает, пишет в файл, печатает ошибку
}
```

Лучше разделить:

- `read_numbers()`
- `calculate_stats()`
- `print_stats()`

## Передача аргументов

### По значению

```cpp
int add_one(int value) {
    return value + 1;
}
```

Функция получает копию.

### По ссылке

```cpp
void increment(int& value) {
    ++value;
}
```

Функция работает с исходным объектом.

### По `const&`

```cpp
void print_name(const std::string& name) {
    std::cout << name << '\n';
}
```

Это очень важный паттерн: не копировать большой объект, если нужно только прочитать его.

## `std::string` и `std::vector`

Это два базовых инструмента junior C++.

### `std::string`

```cpp
std::string text = "hello";
std::cout << text.size() << '\n';
```

### `std::vector`

```cpp
std::vector<int> numbers = {1, 2, 3, 4};
numbers.push_back(5);
```

Почему `vector` важен:

- это нормальный динамический массив из стандартной библиотеки
- он безопаснее и удобнее, чем ручное управление сырыми массивами
- в большинстве junior-задач этого контейнера достаточно

## Базовая обработка ввода

```cpp
int x = 0;
int y = 0;

if (!(std::cin >> x >> y)) {
    std::cerr << "Invalid input\n";
    return 1;
}
```

Это уже лучше, чем молча предполагать, что ввод всегда корректный.

## Типичные ошибки новичка

- все писать в `main()`
- не использовать `const`
- копировать строки и векторы без причины
- давать переменным бессмысленные имена
- игнорировать ошибки ввода
- бояться разбивать код на функции

## Как выглядит уже нормальная маленькая программа

```cpp
#include <iostream>
#include <vector>

int sum(const std::vector<int>& numbers) {
    int result = 0;
    for (int value : numbers) {
        result += value;
    }
    return result;
}

int main() {
    std::vector<int> numbers = {1, 2, 3, 4, 5};
    std::cout << "Sum: " << sum(numbers) << '\n';
    return 0;
}
```

Здесь уже есть:

- отдельная функция
- `const&`
- читаемые имена
- range-based loop

## Что важно вынести из лекции

- C++ программа - это не просто синтаксис, а связка код + компиляция + запуск
- `const`, функции и `std::vector` должны быстро стать привычкой
- задача junior не в том, чтобы писать "умно", а в том, чтобы писать надежно и понятно
