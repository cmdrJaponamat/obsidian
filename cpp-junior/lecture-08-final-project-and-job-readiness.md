# Лекция 8. Финальный проект и готовность к junior-позиции

## Зачем нужен финальный проект

Финальный проект - это не просто "что-то написать". Он нужен, чтобы:

- собрать все навыки вместе
- увидеть свои реальные пробелы
- получить материал для резюме и разговора на интервью

Без проекта знания часто остаются фрагментами.

## Каким должен быть проект

Для junior C++ достаточно проекта, который:

- собирается через CMake
- имеет нормальную структуру
- решает внятную прикладную задачу
- содержит тесты
- имеет README

Тебе не нужен огромный сложный продукт. Нужен законченный, аккуратный и объяснимый проект.

## Какие проекты подходят

- `task-tracker-cli`
- `log-analyzer`
- `file-indexer`
- `mini-chat-server`, если уже есть силы на сеть

Лучше маленький законченный проект, чем амбициозный недострой.

## Что в проекте проверяет рынок

Косвенно работодатель смотрит:

- умеешь ли ты структурировать код
- понимаешь ли ты lifetime и владение ресурсами
- умеешь ли пользоваться STL
- можешь ли собирать и тестировать проект
- умеешь ли описывать запуск и ограничения

## Как строить проект

Полезный порядок работы:

1. описать минимальный scope
2. разложить проект на модули
3. реализовать happy path
4. добавить обработку ошибок
5. добавить тесты
6. вычистить структуру и README

## Что часто идет не так

- scope слишком большой
- нет завершенного результата
- код работает только "у автора на машине"
- README ничего не объясняет
- нет внятного ответа, почему проект устроен именно так

## Как понять, что проект уже можно показывать

Спроси себя:

- проект собирается с нуля по README?
- я могу объяснить каждый модуль?
- есть тесты хотя бы на важную логику?
- обработаны ли очевидные ошибки?
- не стыдно ли мне открыть код перед интервьюером?

Если ответы в целом "да", проект уже полезен.

## Как понять, что ты готов подаваться

Ниже разумный чеклист:

- решаешь базовые задачи на строки, массивы, map/set
- понимаешь `const`, ссылки, RAII, `unique_ptr`
- умеешь собрать проект через CMake
- умеешь пользоваться Git и Linux CLI
- можешь показать хотя бы один завершенный проект
- можешь объяснить, как искал бы баг

Это уже достаточно для первых откликов на стажировки и junior-позиции.

## Что делать после завершения C++ трека

Есть два разумных пути:

1. идти искать junior C++ роли и параллельно усиливать pet-project
2. параллельно качать второй стек через [[../rust-junior/README|Rust Junior Track]]

## Вопросы для самопроверки

1. Почему небольшой законченный проект лучше огромного недостроя?
2. Что ты расскажешь о своем проекте за 2 минуты?
3. Какие пробелы у тебя еще остались?
4. Готов ли ты к первым откликам уже сейчас?
