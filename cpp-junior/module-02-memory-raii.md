# Модуль 2. Память, ссылки, RAII

## Цель

Понять lifetime объектов и перестать писать код, где память "живет как получится".

## Теория

### Темы

- stack vs heap
- указатели и ссылки
- `new/delete` и почему ими лучше не злоупотреблять
- RAII
- `std::unique_ptr`
- `std::shared_ptr` и когда он не нужен
- `std::move`

### Главное правило

Если ресурс можно завернуть в объект с автоматическим освобождением, так и делай.

## Практика

1. Напиши класс `FileLogger`, который открывает файл в конструкторе и закрывает в деструкторе.
2. Сделай класс `IntBuffer`, который хранит динамический массив.
3. Для `IntBuffer` реализуй:
   - конструктор
   - деструктор
   - копирование
   - перемещение
4. Сравни две версии:
   - на raw pointers
   - на `std::vector<int>`

## Что нужно понять руками

- почему rule of three/five вообще существует
- чем move полезен на практике
- почему `shared_ptr` не является "умным решением по умолчанию"

Что произойдет, если скопировать объект, который владеет сырым указателем, но копирующий конструктор не реализован явно?
