# Модуль 3. ООП и STL

## Цель

Научиться писать код в более рабочем стиле: не только функции, но и нормальные модели данных.

## Теория

### STL минимум для junior

- `vector`
- `deque`
- `map`
- `unordered_map`
- `set`
- `optional`
- `variant`
- стандартные алгоритмы: `sort`, `find_if`, `count_if`, `transform`

### ООП минимум

- инкапсуляция
- композиция
- интерфейсы через абстрактные классы
- виртуальные функции
- когда наследование не нужно

## Практика

1. Сделай систему учета книг:
   - `Book`
   - `Library`
   - поиск по названию и автору
2. Реализуй хранение книг через `vector`.
3. Добавь индекс по id через `unordered_map`.
4. Добавь экспорт списка книг в текстовый файл.
5. Найди, где здесь уместна композиция, а где наследование не дает пользы.

## Дополнительная практика

Реши 10 маленьких задач на STL:

- подсчет частот слов
- удаление дублей
- сортировка структур по нескольким полям
- поиск первого элемента по предикату
- группировка строк по первой букве

## Критерий перехода дальше

Ты можешь решить прикладную задачу, не изобретая собственные контейнеры там, где STL уже все дал.
