# Модуль 4. Алгоритмы и сложность

## Цель

Подготовить базу под интервью и научиться выбирать нормальные структуры данных.

## Теория

- `O(1)`, `O(log n)`, `O(n)`, `O(n log n)`, `O(n^2)`
- массив, вектор, список, стек, очередь, хеш-таблица
- бинарный поиск
- two pointers
- sliding window
- prefix sums
- сортировки на уровне понимания

## Практика

Реши минимум по 2 задачи на каждую тему:

- строки
- массивы
- `map/set`
- стек/очередь
- бинарный поиск

## Обязательный набор задач

1. Проверка палиндрома.
2. Two sum.
3. Подсчет частоты слов.
4. Поиск первого повторяющегося символа.
5. Скобочная последовательность.
6. Слияние двух отсортированных массивов.
7. Максимальная сумма подмассива фиксированной длины.
8. Бинарный поиск по ответу на простой задаче.

## Что важно на собеседовании

- сначала назвать структуру данных
- потом оценить сложность
- потом писать код

Не наоборот.
