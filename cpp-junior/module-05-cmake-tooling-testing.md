# Модуль 5. CMake, tooling, tests

## Цель

Перестать быть "человеком с одним `main.cpp`" и перейти к нормальной сборке.

## Теория

### Что нужно уметь

- создать проект на CMake
- подключить несколько исходников
- собрать debug/release
- запускать тесты
- пользоваться `clang-format`
- читать предупреждения компилятора

### Минимальный `CMakeLists.txt` должен уметь

- собирать приложение
- собирать тесты
- включать строгие warning flags

## Практика

1. Возьми любой проект из модулей 1-3.
2. Разбей его на:
   - `src/`
   - `include/`
   - `tests/`
3. Добавь CMake.
4. Подключи хотя бы один тестовый фреймворк:
   - Catch2
   - doctest
5. Напиши минимум 5 тестов.
6. Настрой автоформатирование.

## Проверочный список

- проект собирается одной командой
- код не лежит в одном файле
- есть README с шагами сборки
- тесты запускаются отдельно

## Результат

После этого модуля любой твой учебный проект должен выглядеть как маленький инженерный проект, а не как черновик.
