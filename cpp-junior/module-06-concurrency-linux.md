# Модуль 6. Потоки и Linux

## Цель

Получить уровень, достаточный для базового разговора о системной разработке и утилитах под Linux.

## Теория

### Темы

- процесс и поток
- `std::thread`
- `mutex`
- `lock_guard`
- race condition
- deadlock на уровне понимания
- файлы, пайпы, stdout/stderr
- аргументы командной строки

## Практика

1. Напиши программу, которая считает статистику по большому файлу в несколько потоков.
2. Напиши логгер с защитой через mutex.
3. Сделай CLI-утилиту:
   - принимает путь к файлу
   - считает количество строк, слов, байт
   - выводит результат как упрощенный `wc`

## Важно

Не пытайся сразу лезть в lock-free и сложную конкурентность. Для junior достаточно понимать базовые риски и аккуратно пользоваться примитивами.

## Вопросы для самопроверки

- Почему вывод из нескольких потоков может ломаться?
- Чем `lock_guard` лучше ручного `lock/unlock`?
