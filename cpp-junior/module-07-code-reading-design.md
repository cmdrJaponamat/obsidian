# Модуль 7. Чтение кода и дизайн

## Цель

Научиться читать чужой код и принимать разумные структурные решения без лишней "архитектуры ради архитектуры".

## Теория

- single responsibility
- слабая связанность
- высокая связность внутри модуля
- интерфейс vs реализация
- dependency injection на простом уровне
- anti-pattern: god object

## Практика

1. Возьми свой старый проект и найди:
   - слишком длинные функции
   - дублирование
   - смешение ввода/бизнес-логики/вывода
2. Перепиши его на модули.
3. Добавь интерфейс для хранилища:
   - память
   - файл
4. Подготовь короткое объяснение:
   - почему именно так разбил проект
   - какие компромиссы принял

## Собеседовательная часть

Потренируйся устно отвечать:

- что такое RAII
- чем `vector` отличается от `list`
- когда использовать `unique_ptr`
- что делает `virtual`
- зачем нужен CMake

## Критерий готовности

Ты способен открыть небольшой чужой проект и за 20-30 минут понять, где точка входа, как собирается код и где вероятны ошибки.
