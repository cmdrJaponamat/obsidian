# Модуль 8. Финальный проект

## Цель

Собрать портфельный проект, который можно показать и по которому удобно проходить интервью.

## Варианты проекта

Выбери один:

1. `task-tracker-cli`
2. `log-analyzer`
3. `file-indexer`
4. `mini-chat-server` без GUI, если готов к усложнению

## Обязательные требования

- CMake
- несколько модулей
- тесты
- работа с файлами
- README
- понятная модель данных
- аккуратная обработка ошибок

## Желательные требования

- конфиг
- парсинг аргументов
- несколько режимов запуска
- profiling/debug notes

## Что показать мне на проверку

- структуру проекта
- ключевые `.h/.cpp`
- `CMakeLists.txt`
- README
- список известных ограничений

## Финальная самопроверка

- Могу ли я объяснить каждый класс?
- Понимаю ли я, где и кто владеет ресурсами?
- Есть ли в коде очевидные места для упрощения?
- Смогу ли я устно защитить этот проект на интервью?
