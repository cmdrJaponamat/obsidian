# Курс: CMake для junior C++

## Почему это нужно рынку

Для junior C++ под Linux CMake встречается очень часто. Если кандидат пишет код, но не может собрать его как проект, это сильный минус.

## Цель

Довести сборку до уровня "я умею поднять нормальный небольшой проект, тесты и конфигурации debug/release".

## Теория

### Основа

- `project()`
- `add_executable()`
- `add_library()`
- `target_include_directories()`
- `target_link_libraries()`
- `target_compile_features()`

### Хорошие практики

- target-based CMake
- не писать глобальные include и compile flags без причины
- separate `src/`, `include/`, `tests/`
- `CMAKE_EXPORT_COMPILE_COMMANDS`

### Тесты

- `enable_testing()`
- `add_test()`
- `ctest`

## Практика

1. Создай проект:
   - библиотека с бизнес-логикой
   - CLI-приложение
   - тесты
2. Настрой сборку через Ninja.
3. Добавь `Debug` и `Release`.
4. Подключи тесты.
5. Проверь, что `clangd` видит проект.

## Что нужно уметь после курса

- быстро создавать новый C++ проект
- подключать новые файлы без хаоса
- запускать тесты и отладку
- объяснить, как проект собирается

## Mini challenge

Возьми старый учебный C++ проект и переведи его из "одного файла" в структуру с библиотекой, приложением и тестами.
