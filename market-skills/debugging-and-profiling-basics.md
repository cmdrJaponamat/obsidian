# Курс: Отладка и базовый profiling

## Почему это нужно рынку

Вакансии C++ под Linux часто прямо называют `gdb`, `valgrind`, `perf`, `strace`. Для junior достаточно не экспертности, а рабочего базового уровня.

## Цель

Научиться не гадать, а исследовать поведение программы инструментами.

## Теория

### Отладка

- debug build
- breakpoint
- step into / step over
- просмотр переменных
- backtrace

### Инструменты

- `gdb`
- `lldb`
- `valgrind` для C++
- `perf` на уровне знакомства
- `strace` для системных вызовов

### Что искать

- падения
- use-after-free и утечки
- бесконечные циклы
- неожиданные системные вызовы
- узкие места по времени

## Практика

1. Напиши программу с намеренным багом.
2. Найди причину падения через `gdb`.
3. Для C++ проверь проект через `valgrind`.
4. Сними базовый `perf`-репорт на горячую функцию.
5. Через `strace` посмотри, какие файлы реально открывает программа.

## Что нужно уметь после курса

- получить stack trace
- локализовать место сбоя
- отличать предположение от подтвержденного факта
- хотя бы базово объяснить, где программа тратит время
