# Курс: Git для разработчика

## Почему это нужно рынку

Git стабильно требуется в junior C++ и Rust вакансиях. На практике это один из самых дешевых навыков для работодателя и один из самых заметных маркеров "человек уже работал как разработчик".

## Цель

Научиться не просто делать `git add .`, а работать как человек, который не ломает историю и может поддерживать нормальный поток изменений.

## Теория

### База

- `git init`
- `git status`
- `git add`
- `git commit`
- `git log`
- `.gitignore`

### Рабочий цикл

- ветки
- `git switch`
- `git branch`
- `git merge`
- `git rebase` на уровне понимания
- разрешение простых конфликтов

### Диагностика и аккуратность

- `git diff`
- `git restore` и где с ним осторожно
- `git stash`
- `git blame`
- как читать историю проекта

## Практика

1. Создай учебный репозиторий.
2. Сделай 10 осмысленных коммитов вместо одного большого.
3. Смоделируй feature branch:
   - создай ветку
   - внеси изменение
   - влей обратно
4. Искусственно создай merge conflict и реши его.
5. Добавь `.gitignore` для C++ и Rust проектов.
6. Попрактикуйся читать историю:
   - найди, какой коммит внес конкретную строку
   - объясни, что он менял

## Что нужно уметь после курса

- оформить понятную историю коммитов
- не бояться веток
- понимать, что происходит при merge/rebase
- не терять изменения случайно

## Мини-проект

Возьми любой учебный проект из языковых треков и оформи весь его прогресс через Git так, чтобы по истории было видно этапы работы.
