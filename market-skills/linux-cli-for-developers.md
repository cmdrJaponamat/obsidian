# Курс: Linux CLI для разработчика

## Почему CLI нельзя считать вторичным навыком

Для C++ и Rust под Linux командная строка - это не “режим для фанатов терминала”, а нормальная рабочая среда.

Через CLI ты:

- ищешь файлы
- собираешь проект
- читаешь логи
- запускаешь тесты
- управляешь процессами

Если делать все это только через GUI, скорость и управляемость работы падают.

## Файлы и навигация

Базовые команды:

- `pwd`
- `ls`
- `cd`
- `cp`
- `mv`
- `mkdir`

Важно не просто знать их названия, а понимать, что ты должен уметь быстро ориентироваться в дереве проекта.

## Поиск

### `find`

Нужен для поиска файлов по имени, типу и другим условиям.

### `rg`

`ripgrep` нужен для быстрого поиска текста по проекту.

На практике `rg` - один из самых полезных инструментов разработчика.

Если надо найти:

- где используется функция
- где встречается TODO
- где упоминается конкретный класс

то обычно это делается через `rg`.

## Просмотр текста и логов

Полезный минимум:

- `cat`
- `less`
- `head`
- `tail`

Почему это важно:

- не всегда нужен редактор
- иногда нужно быстро увидеть хвост лога или начало файла

## Перенаправление

Операторы:

- `>`
- `>>`
- `2>`

позволяют направлять:

- обычный вывод
- ошибки
- и дописывать в файл или перезаписывать его

Это важно для:

- логов сборки
- диагностики ошибок
- автоматизации

## Процессы

Нужно понимать хотя бы базово:

- `ps`
- `top` или `htop`
- `kill`

Программа может:

- зависнуть
- крутиться в фоне
- занимать ресурсы

CLI дает над этим контроль.

## Переменные окружения

Окружение влияет на:

- пути поиска
- конфигурацию программ
- режимы запуска

Это особенно важно для:

- сборки
- конфигов
- отладки

## Что нужно вынести из курса

- Linux CLI - это основной рабочий инструмент, а не запасной режим
- `rg`, `find`, `less`, `ps`, `kill` должны стать привычными
- перенаправление и окружение - часть ежедневной инженерной практики

## Практика

1. Найди все `.cpp` или `.rs` файлы в каталоге.
2. Найди все места с `TODO`.
3. Сохрани лог сборки в файл.
4. Запусти процесс в фоне и останови его.
5. Через CLI собери и протестируй проект.
