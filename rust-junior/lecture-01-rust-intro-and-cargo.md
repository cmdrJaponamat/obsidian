# Лекция 1. Что такое Rust, как устроен Cargo и из чего состоит программа

## Как читать этот курс

Эта лекция написана для старта с нуля. Я не буду предполагать, что ты уже "чувствуешь" модель Rust. Наоборот: задача в том, чтобы шаг за шагом построить правильную ментальную модель.

## Почему Rust вообще существует

Rust появился не просто как "еще один язык". У него довольно конкретная цель:

- давать хороший контроль над производительностью
- помогать писать безопасный код без сборщика мусора
- ловить целый класс ошибок еще на этапе компиляции

Очень грубо можно сказать так:

- C и C++ дают много свободы, но требуют высокой дисциплины
- Rust тоже дает контроль, но пытается заставить тебя выражать инварианты явно

Из-за этого Rust сначала кажется более строгим. Зато потом многие ошибки обнаруживаются до запуска.

## Где Rust нужен на рынке

Rust встречается в:

- системном программировании
- backend и сетевых сервисах
- CLI-инструментах
- инфраструктурных утилитах
- проектах, где важны надежность и предсказуемость

Для junior сейчас важнее не "знать все crate в мире", а понимать:

- как создать и запустить проект
- как устроены базовые типы и функции
- почему ownership - не случайная странность, а центральная идея языка

## Что такое Cargo

Cargo - это стандартный инструмент Rust, который решает сразу несколько задач:

- создание проекта
- сборка
- запуск
- тестирование
- управление зависимостями

Это одна из самых сильных сторон экосистемы Rust: базовый workflow стандартизирован.

## Что дает стандартный workflow

Когда у тебя есть единый инструмент, это означает:

- проект проще перенести на другую машину
- другому разработчику проще понять, как все запускается
- меньше "самописной" инфраструктуры вокруг мелких проектов

В C++ часто приходится отдельно собирать workflow из компилятора, CMake, тестов и разных инструментов. В Rust на базовом уровне это уже собрано вокруг Cargo.

## Создание проекта

Типичный старт:

```bash
cargo new calc_rs
```

После этого появляется структура проекта, где уже есть:

- `Cargo.toml`
- `src/main.rs`

## Что такое `Cargo.toml`

Это основной файл конфигурации проекта.

Он описывает:

- имя проекта
- версию
- зависимости
- настройки пакета

В дальнейшем ты будешь регулярно в него добавлять библиотеки, например:

- `serde`
- `tokio`
- `clap`

## Что такое `src/main.rs`

Это файл с точкой входа бинарного приложения.

Минимальная программа:

```rust
fn main() {
    println!("Hello, Rust!");
}
```

## Что здесь происходит

### `fn`

Так объявляется функция.

### `main`

Это специальная функция, с которой начинается выполнение программы.

### `println!`

Это макрос, а не обычная функция. Макросы в Rust отмечаются `!`.

На старте достаточно помнить:

- `println!` выводит строку в консоль
- макрос в Rust может принимать гибкий синтаксис

Не нужно пока углубляться в устройство макросов.

## Основные команды Cargo

### `cargo run`

Собирает и запускает проект.

### `cargo check`

Проверяет код быстрее, чем полная сборка.

Это очень полезная команда в повседневной работе. Ей стоит пользоваться часто.

### `cargo test`

Запускает тесты.

### `cargo fmt`

Форматирует код.

## Почему `cargo check` так полезен

На старте у тебя будет много ситуаций, когда код еще не готов к запуску, но ты уже хочешь понять:

- правильно ли написаны типы
- не ругается ли компилятор
- не сломал ли ты сигнатуры и вызовы

`cargo check` помогает быстро получать обратную связь.

## Переменные и `mut`

В Rust переменные по умолчанию неизменяемые.

```rust
let x = 10;
```

Если нужно менять значение:

```rust
let mut counter = 0;
counter += 1;
```

## Почему переменные по умолчанию неизменяемы

Это не каприз языка. Это часть общей философии:

- изменение состояния должно быть видно явно
- меньше случайных модификаций
- проще понимать, где данные живут как константы, а где реально меняются

То есть `mut` - это не "дополнительная помеха", а сигнал о намерении.

## Типы

В Rust типы играют очень важную роль.

Базовый минимум:

- `i32`, `i64`, `usize`
- `f64`
- `bool`
- `char`
- `String`
- `&str`

Пример:

```rust
let age: i32 = 21;
let ratio: f64 = 3.14;
let ready: bool = true;
let letter: char = 'A';
```

## Что такое `String` и `&str`

Это одна из первых тем, которую новички часто чувствуют смутно.

### `&str`

Это строковый срез, то есть ссылка на строковые данные.

Пример:

```rust
let name: &str = "Ivan";
```

Здесь строковый литерал живет в памяти программы, а переменная ссылается на него.

### `String`

Это владеющий строковый тип.

```rust
let mut text = String::from("hello");
text.push('!');
```

Он:

- владеет строковыми данными
- может их изменять
- управляет своей памятью

### Практическая разница

Если говорить грубо:

- `&str` удобно, когда нужно просто смотреть на строку
- `String` нужен, когда ты хочешь владеть строкой и изменять ее

Позже ты увидишь, что это напрямую связано с ownership.

## Выражения и инструкции

Rust любит выражения. Это важная stylistic и semantic особенность языка.

Пример:

```rust
let value = if true { 1 } else { 2 };
```

Здесь `if` возвращает значение.

Это сначала может удивлять, но потом делает код выразительным.

## `if`, `loop`, `while`, `for`

### `if`

```rust
if age >= 18 {
    println!("adult");
} else {
    println!("minor");
}
```

### `loop`

```rust
let mut count = 0;
loop {
    count += 1;
    if count == 3 {
        break;
    }
}
```

### `while`

```rust
let mut x = 3;
while x > 0 {
    println!("{x}");
    x -= 1;
}
```

### `for`

```rust
for value in [1, 2, 3] {
    println!("{value}");
}
```

На старте `for` особенно полезен, потому что он лучше вписывается в безопасную модель обхода коллекций.

## Функции

```rust
fn square(value: i32) -> i32 {
    value * value
}
```

Здесь важно заметить:

- тип аргумента указывается явно
- тип результата тоже указывается явно
- последнее выражение можно вернуть без `return`

## Почему строгие сигнатуры полезны

В Rust типы в сигнатурах функций - это часть общения с будущим читателем кода.

Когда ты видишь:

```rust
fn parse_number(text: &str) -> i32
```

ты сразу понимаешь:

- функция принимает строковый срез
- возвращает целое число

Позже мы увидим, что для реального парсинга сигнатура должна быть честнее и возвращать `Result`, но базовый принцип уже видно: интерфейс функции должен явно говорить о поведении.

## Что такое `match`

`match` - один из центральных инструментов Rust.

```rust
let number = 2;

match number {
    1 => println!("one"),
    2 => println!("two"),
    _ => println!("other"),
}
```

Он важен не просто как "switch", а как способ явно разбирать варианты состояний.

Позже он будет особенно полезен для:

- `Option`
- `Result`
- пользовательских `enum`

## Почему компилятор Rust кажется строгим

Потому что он проверяет больше инвариантов заранее.

На старте это раздражает. Но в этом и есть ценность языка: многие потенциальные проблемы не доживают до запуска.

Правильное отношение:

- не бороться с компилятором
- а пытаться понять, какой инвариант он защищает

## Типичные ошибки новичка

- путать `String` и `&str`
- раздражаться на `mut` вместо понимания, зачем он нужен
- бояться текста ошибок компилятора
- пытаться сразу понять весь язык целиком

## Что нужно вынести из лекции

- Rust - это язык, где типы и правила владения выражаются очень явно
- Cargo задает стандартный рабочий процесс
- `String` и `&str` - это не одно и то же
- неизменяемость по умолчанию - часть общей дисциплины языка
