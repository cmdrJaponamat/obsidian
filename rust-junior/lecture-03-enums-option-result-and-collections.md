# Лекция 3. Enum, Option, Result и коллекции

## Зачем это изучать

Rust силен тем, что заставляет выражать состояние программы явно. Вместо размытых договоренностей ты работаешь с типами, которые показывают, что может произойти.

## `enum`

`enum` позволяет описать набор возможных вариантов.

```rust
enum Command {
    Add(String),
    List,
    Done(usize),
}
```

Это удобно потому, что команда не хранится как "строка, которую потом как-нибудь поймем", а моделируется типом.

## `Option`

Когда значение может отсутствовать, вместо `null` используется `Option`.

```rust
fn find_name(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Ivan"))
    } else {
        None
    }
}
```

Это заставляет вызывающий код явно обработать отсутствие значения.

## `Result`

Когда операция может завершиться ошибкой, используется `Result<T, E>`.

```rust
fn parse_number(text: &str) -> Result<i32, std::num::ParseIntError> {
    text.parse::<i32>()
}
```

Плюс такой модели:

- успех и ошибка выражены в типе
- нельзя "забыть", что ошибка возможна

## `match` как инструмент разбора состояний

```rust
match parse_number("42") {
    Ok(value) => println!("parsed: {value}"),
    Err(error) => println!("error: {error}"),
}
```

Это естественный способ работы с вариантами состояния.

## `if let`

Когда нужен только один интересующий вариант:

```rust
if let Some(name) = find_name(1) {
    println!("{name}");
}
```

Это короче и читаемее полного `match`, если остальные случаи неинтересны.

## Коллекции

### `Vec<T>`

Основной список элементов.

```rust
let mut numbers = vec![1, 2, 3];
numbers.push(4);
```

### `HashMap<K, V>`

Основной словарь.

```rust
use std::collections::HashMap;

let mut counts = HashMap::new();
counts.insert(String::from("rust"), 1);
```

## Итераторы на базовом уровне

В Rust часто работа идет не через ручные индексы, а через методы итераторов.

Пример:

```rust
let values = vec![1, 2, 3, 4];
let sum: i32 = values.iter().sum();
```

Пока не надо знать все тонкости. Достаточно понимать:

- `iter()` дает чтение
- `iter_mut()` дает изменяемый доступ
- `into_iter()` часто передает владение

## Почему это важно для рынка

На junior Rust позиции почти всегда полезно уметь:

- читать код с `Option` и `Result`
- моделировать команды и состояния через `enum`
- пользоваться `Vec` и `HashMap`
- не паниковать от `match`

## Типичные ошибки новичка

- использовать `unwrap()` везде подряд
- моделировать команды строками вместо `enum`
- не обрабатывать `None` и `Err`
- путаться в том, какой итератор что делает

## `unwrap()` и почему с ним осторожно

`unwrap()` удобен в учебных или строго контролируемых местах, но он приводит к panic, если значение отсутствует или есть ошибка.

На практике безопаснее:

- обработать `Result`
- обработать `Option`
- вернуть ошибку вверх
