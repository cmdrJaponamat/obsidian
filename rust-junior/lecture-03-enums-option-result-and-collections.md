# Лекция 3. Enum, Option, Result и коллекции: как Rust заставляет моделировать мир явно

## Почему эта тема глубже, чем кажется

В ряде языков типичная программа строится на скрытых договоренностях:

- если функция вернула `-1`, значит не нашла
- если строка пустая, значит произошла ошибка
- если значение `null`, то "ну как-нибудь обработаем"

Rust старается вытеснить такой стиль. Вместо неявных соглашений он подталкивает выражать состояние типами.

## `enum`: не просто перечисление

`enum` в Rust - это не только аналог простого перечисления констант. Это способ описать набор различных вариантов состояния, каждый из которых может нести свои данные.

Пример:

```rust
enum Command {
    Add(String),
    List,
    Done(usize),
}
```

Это значит:

- команда может быть `Add` и содержать строку
- или `List` без дополнительных данных
- или `Done` с номером задачи

То есть тип прямо описывает форму входа.

## Почему это лучше строки

Если команду передавать как строку:

- легко опечататься
- логика разбросана по `if`/`else`
- часть случаев может быть забыта

Если команда представлена `enum`, компилятор помогает проверять разбор вариантов.

## `Option<T>`

`Option<T>` означает:

- `Some(T)` - значение есть
- `None` - значения нет

Это не просто "обертка". Это способ сказать:

- отсутствие значения является нормальным возможным состоянием

Пример:

```rust
fn find_name(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Ivan"))
    } else {
        None
    }
}
```

## Почему `Option` лучше скрытого отсутствия

Если функция возвращает `String`, но "при неудаче" кладет пустую строку, у тебя появляется двусмысленность:

- это реально пустая строка
- или специальный код отсутствия

`Option` снимает эту двусмысленность.

## `Result<T, E>`

`Result` нужен, когда операция может не удаться.

Он имеет два варианта:

- `Ok(T)`
- `Err(E)`

Пример:

```rust
fn parse_number(text: &str) -> Result<i32, std::num::ParseIntError> {
    text.parse::<i32>()
}
```

## Почему `Result` фундаментален

Он делает ошибку частью контракта функции.

Когда ты видишь сигнатуру:

```rust
fn load_config() -> Result<Config, ConfigError>
```

ты уже знаешь:

- функция может вернуть конфиг
- а может вернуть конкретную ошибку

Это намного честнее, чем падение через panic или магическое значение.

## `match`

`match` нужен не просто как синтаксическая игрушка, а как основной способ разбирать варианты состояния.

```rust
match parse_number("42") {
    Ok(value) => println!("parsed: {value}"),
    Err(error) => println!("error: {error}"),
}
```

Почему он полезен:

- явно показывает все рассматриваемые случаи
- хорошо сочетается с `enum`, `Option`, `Result`

## `if let`

Иногда тебе важен только один вариант.

```rust
if let Some(name) = find_name(1) {
    println!("{name}");
}
```

Это сокращенная форма, когда не нужен полный разбор всех веток.

## `unwrap()`

`unwrap()` - это утверждение:

- "я уверен, что здесь все хорошо"

Если уверенность неверна, программа падает.

### Когда `unwrap()` допустим

- в очень маленьких экспериментах
- в тестах
- в местах, где инвариант уже гарантирован и это действительно осознанное решение

### Когда `unwrap()` плох

- в пользовательском коде
- в CLI, который должен сообщать ошибки нормально
- в коде, где ошибка ожидаема как часть жизни программы

## Коллекции в Rust

## `Vec<T>`

Это основной контейнер последовательных данных.

```rust
let mut values = vec![1, 2, 3];
values.push(4);
```

Когда подходит:

- нужен список
- важен порядок
- нужен проход по всем элементам

## `HashMap<K, V>`

Это словарь.

```rust
use std::collections::HashMap;

let mut counts = HashMap::new();
counts.insert(String::from("rust"), 1);
```

Когда подходит:

- нужен доступ по ключу
- нужно считать частоты
- нужно хранить индексы и сопоставления

## Почему `HashMap` нужен для частот, а не `HashSet`

Это тот же смысл, что и в C++:

- `HashSet` хранит только факт наличия
- `HashMap` хранит ключ и связанное значение

Для частоты нужен именно счетчик, а значит нужна map-like структура.

## Итераторы

В Rust важно понимать не только циклы, но и базовые виды итерации.

### `iter()`

Читает элементы по ссылке.

### `iter_mut()`

Дает изменяемый доступ к элементам.

### `into_iter()`

Часто передает владение элементами дальше.

Эта разница важна, потому что итераторы тесно связаны с ownership.

## Что нужно вынести из лекции

- `enum` описывает набор вариантов состояния, а не только список констант
- `Option` честно выражает отсутствие значения
- `Result` честно выражает возможность ошибки
- `match` и `if let` - естественные инструменты разбора таких состояний
- `Vec` и `HashMap` - базовые рабочие контейнеры Rust
