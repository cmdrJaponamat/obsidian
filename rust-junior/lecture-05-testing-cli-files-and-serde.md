# Лекция 5. Тесты, CLI, файлы и serde

## Почему эта тема рыночная

Многие junior Rust задачи на практике выглядят так:

- CLI-приложение
- чтение аргументов
- работа с файлами
- сериализация данных
- тесты

Это уже похоже на полезный рабочий код, а не только на учебные функции.

## Тесты в Rust

Rust хорошо поддерживает тестирование из коробки.

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn adds_numbers() {
        assert_eq!(add(2, 3), 5);
    }
}
```

Это удобно потому, что ты можешь быстро проверять поведение при изменениях.

## Unit tests и integration tests

### Unit tests

Тестируют маленький кусок логики.

### Integration tests

Проверяют, как работает проект как единое целое.

На junior уровне полезно уметь и то, и другое.

## CLI

Минимальный CLI можно делать через `std::env`, но в реальных проектах часто полезен `clap`.

Даже если сначала делаешь проще, важно понимать:

- какие команды поддерживаются
- как валидируются аргументы
- как выглядят сообщения об ошибках

## Работа с файлами

Нужно уметь:

- читать файл
- писать файл
- понимать, что операция может завершиться ошибкой

Пример идеи:

- загрузить задачи из JSON
- изменить состояние
- сохранить обратно

## `serde` и `serde_json`

`serde` - стандартный выбор для сериализации и десериализации в Rust.

Пример:

```rust
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize)]
struct Task {
    title: String,
    done: bool,
}
```

Это позволяет хранить данные в JSON без ручного парсинга.

## Ошибки

При работе с файлами и вводом ошибки - это норма.

Важно не "делать вид, что их нет", а обрабатывать их:

- возвращать `Result`
- давать понятные сообщения
- не использовать `unwrap()` там, где ошибка возможна в реальной жизни

## Что значит законченный CLI-проект

- имеет понятные команды
- может работать с файлами
- ошибки обрабатываются осмысленно
- есть тесты
- есть README

## Типичные ошибки новичка

- логика, ввод и файловые операции перемешаны в `main`
- тестов почти нет
- все ошибки через `unwrap()`
- формат хранения не задокументирован
