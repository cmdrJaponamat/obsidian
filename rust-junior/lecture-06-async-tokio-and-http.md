# Лекция 6. Async, Tokio и HTTP

## Почему это важно для рынка

Во многих Rust-вакансиях даже на junior+1 или intern+ уровнях всплывают:

- `tokio`
- `async/await`
- HTTP-клиенты и сервисы
- работа с JSON API

Поэтому нужно хотя бы базовое рабочее понимание.

## Что такое async

Async позволяет описывать операции, которые могут ждать:

- сеть
- диск
- таймеры

Вместо блокировки потока можно позволить runtime выполнять другие задачи.

## `async/await` на интуитивном уровне

Если кратко:

- `async fn` возвращает future
- `await` ждет результат этой future

Но сама по себе future не исполняется без runtime.

## Зачем нужен runtime

Например, `tokio`:

- управляет выполнением async-задач
- дает сетевые примитивы, таймеры и другие инструменты

Без runtime async-код не "оживает" сам.

## Когда async уместен

Чаще всего:

- сетевые запросы
- высококонкурентные I/O-задачи
- сервисы и обработчики запросов

Когда не нужен:

- простая CPU-bound логика
- маленькие программы, где блокирующий код проще и понятнее

Не надо ставить `async` везде подряд.

## HTTP-клиент как типичный кейс

Очень полезный учебный сценарий:

1. сходить по API
2. получить JSON
3. десериализовать
4. отфильтровать данные
5. сохранить результат

Это сразу соединяет:

- async
- HTTP
- `serde`
- ошибки
- файловые операции

## Таймауты и ошибки

Сетевой код без таймаутов и нормальной обработки ошибок быстро становится ненадежным.

Нужно помнить:

- сеть может не ответить
- JSON может быть другого формата
- соединение может обрываться

Значит, ошибка - нормальный сценарий, а не исключение из мира.

## Почему здесь полезны базовые сети

Чтобы понимать:

- что запрос уходит по сети
- что HTTP не равен TCP, но работает поверх него
- почему бывают таймауты, DNS-ошибки и проблемы соединения

## Типичные ошибки новичка

- использовать async везде без причины
- смешивать сетевую логику, парсинг и CLI в одном месте
- не ставить таймауты
- игнорировать ошибочные статусы ответа

## Что должен уметь junior после этой темы

- объяснить, зачем нужен runtime
- написать простую async CLI-программу
- получить JSON из HTTP API
- обработать ошибку без panic
