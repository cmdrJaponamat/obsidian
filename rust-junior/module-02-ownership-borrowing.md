# Модуль 2. Ownership и borrowing

## Цель

Понять основу языка. Без этого дальше идти бессмысленно.

## Теория

### Ключевые идеи

- у значения есть владелец
- в один момент времени либо один mutable borrow, либо несколько immutable
- move по умолчанию
- ссылки не должны переживать владельца

### Что не надо делать

- зубрить lifetimes раньше времени
- лечить все через `.clone()`

## Практика

1. Напиши функции:
   - берущую `String` по значению
   - берущую `&String`
   - берущую `&str`
2. Проверь, что происходит после вызова каждой из них.
3. Реализуй структуру `TaskList`:
   - хранит задачи
   - умеет добавлять
   - умеет возвращать список
   - умеет искать по подстроке
4. Сделай это без лишних `clone()`.

## Что нужно понять руками

- почему borrow checker ругается
- какой инвариант он защищает
- как переписать код так, чтобы модель владения была ясной
