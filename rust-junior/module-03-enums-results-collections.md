# Модуль 3. Enum, Result, коллекции

## Цель

Понять, как в Rust моделируются состояния и ошибки без хаоса из `null` и неявных соглашений.

## Теория

- `enum`
- `Option`
- `Result`
- `match`
- `if let`
- `Vec`
- `HashMap`
- iterator basics

## Практика

1. Напиши парсер строки в команду:
   - `add text`
   - `done 3`
   - `list`
2. Смоделируй команды через `enum`.
3. Все ошибки парсинга возвращай через `Result`.
4. Напиши программу подсчета частоты слов через `HashMap`.

## Цель руками

Научиться писать код, где некорректные состояния не размываются по программе, а явно моделируются типами.

## Критерий перехода

Ты спокойно читаешь сигнатуры с `Result<T, E>` и не паникуешь от `match`.
