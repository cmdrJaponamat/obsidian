# Модуль 6. Async и экосистема

## Цель

Познакомиться с тем, что часто встречается в реальных Rust-вакансиях: async, HTTP, конфиги, экосистема.

## Теория

- что такое `async/await`
- зачем нужен runtime
- `tokio`
- базовый HTTP клиент
- `serde`
- env/config

## Практика

1. Напиши CLI, который ходит по HTTP API.
2. Сценарий:
   - получает данные
   - десериализует JSON
   - фильтрует
   - сохраняет результат в файл
3. Сделай async-версию на `tokio`.
4. Добавь таймауты и обработку ошибок.

## Важно

Для junior достаточно понимать:

- почему blocking и async отличаются
- почему нельзя бездумно ставить `async` везде
- как выглядит нормальная ошибка в сетевом коде
